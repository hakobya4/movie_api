Movie API

Essential Features

Business Logic URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None A JSON object holding data about all the movies
Return data (description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user /movies/:title GET None A JSON object holding data about a single movie, containing an id, title, descripion, genre, director(name, bio, birth year, death year) and image URL. Example: 
  {
    id: 1,
    title: 'The Shawshank Redemption',
    descrition: 'Over the course of several years, two 
                convicts form a friendship, seeking consolation 
                and,eventually, redemption through basic compassion.',
    genre:'Drama',
    director: {
        title: 'Frank Darabont',
        bio: 'Frank Árpád Darabont is a French-born American film director, 
              screenwriter and producer.',
        birth_year: 1959,
        death_year:'N/A',
        },
    image_URL: 
        'https://m.media-amazon.com/images/I/51zUbui+gbL._AC_UF894,1000_QL80_.jpg',
    },
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/:title/genre GET None A JSON object holding data about a genre(s) of a single movie, containing genre(s). Example:

genre:'Drama'
Return data about a director (bio, birth year, death year) by name /movies/:title/director GET None A JSON object holding data about a director of a single movie, containing name, bio, birth year, and death year. Example: 
  director: {
    title: 'Frank Darabont',
    bio: 'Frank Árpád Darabont is a French-born 
          American film director, 
          screenwriter and producer.',
    birth_year: 1959,
    death_year:'N/A',
  }
Allow new users to register; /register POST A JSON object holding data about the user to add, structured like: 
{
  username: "JoeJane",
  password: "Password1!",
  email: "example@gmail.com",
}
A JSON object holding data about the user that was added, including an ID: 
{
  id:1,
  username: "JoeJane",
  password: "Password1!",
  email: "example@gmail.com",
}
Allow users to update their user info (username, password, email, date of birth) /:username PUT A JSON object holding data about the updated user information, structured like: 
{
  username: "New Username",
  password: "New Password",
  email: "New Email",
}
A text message indicating whether data of the user has been updated and
a JSON object holding data about the user that was changed: 
{
  New User Information:
  username: "New Username",
  password: "New Password",
  email: "New Email",
}
