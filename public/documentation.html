Movie API

Essential Features

Business Logic URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None A JSON object holding data about all the movies
Return data (description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user /movies/:title GET None A JSON object holding data about a single movie, containing an id, title, descripion, genre, director(name, bio, birth year, death year) and image URL.
Return data about a single genre including name and description /genre/:genre GET None A JSON object holding data about info regarding a single genre.
Return data about movies using a single genre /movies/:genre/genre GET None A JSON object holding data about info regarding movie(s) of a single genre.
Return data about a director (bio, birth year, death year) by movie name /movies/:title/director GET None A JSON object holding data about a director of a single movie, containing name, bio, birth year, and death year.
Return data about a single director (bio, birth year, death year) by director name /director/:director GET None A JSON object holding data about info regarding a single genre.
Allow new users to register; /users POST A JSON object holding data about the user to add,
including username, password, email and favorites,
structured like: 
{
  username: "JoeJane",
  password: "Password1!",
  email: "example@gmail.com",
  favorites: [MovieID: 774747747, 948484884]
}
A JSON object holding data about the user that was added, including an ID: 
{
  id:1,
  username: "JoeJane",
  password: "Password1!",
  email: "example@gmail.com",
  favorites: [MovieID: 774747747, 948484884]
}
Return data about a single user by username /users/:username GET None A JSON object holding data about info on one user.
Allow users to update their user info (username, password, email, date of birth) /user/:username/update PUT A JSON object holding data about the updated user information,
structured like: 
{
  username: "New Username",
  password: "New Password",
  email: "New Email",
}
A text message indicating whether data of the user has been updated and
a JSON object holding data about the user that was changed: 
{
  username: "New Username",
  password: "New Password",
  email: "New Email",
  birthday: "New Birthday"
}
Allow users to add a movie to their list of favorites /users/:username/movies/:MovieID POST None A JSON object holding data about the user who's favorite move list
has changed. Example: 
{
  username: "Username",
  password: "Password",
  email: "Email",
  birthday: "Birthday"
  favories: [new MovieID]
}
Allow users to remove a movie from their list of favorites /users/:username/movies/:MovieID DELETE None A JSON object holding data about the user who's favorite move list
has changed by delition. Example: 
{
  username: "Username",
  password: "Password",
  email: "Email",
  birthday: "Birthday"
  favories: [changed MovieID]
}
Return data about the favorite movie list of one user by name /users/:username/favorites GET None A JSON object holding data about all MovieIDs in an array of favorites
Allow existing users to deregister /users DELETE None A text message indicating whether user has been deleted
